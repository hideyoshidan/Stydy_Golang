# Golang

## Goroutine習得課題

### [x]課題1: 基本的なgoroutineの起動
複数のgoroutineを起動し、それぞれが異なるメッセージを出力するプログラムを作成してください。
- 5つのgoroutineを起動
- それぞれが自分の番号とメッセージを出力
- `time.Sleep()`で実行順序の違いを確認

### [x]課題2: channelを使った通信
goroutine間でデータをやり取りするプログラムを作成してください。
- 送信用goroutineと受信用goroutineを作成
- channelを使って10個の数値を送受信
- すべてのデータを受信後に終了

### [x]課題3: WaitGroupを使った同期
複数のgoroutineの完了を待つプログラムを作成してください。
- `sync.WaitGroup`を使用
- 5つの並行タスクを実行
- すべてのタスク完了後にメッセージを出力

### []課題4: バッファ付きchannelの活用
バッファ付きchannelを使って、プロデューサー・コンシューマーパターンを実装してください。
- バッファサイズ3のchannelを作成
- 生産者goroutineが10個のアイテムを生成
- 消費者goroutineが処理

### []課題5: selectを使った複数channel制御
複数のchannelから同時に受信するプログラムを作成してください。
- 2つ以上のchannelを使用
- `select`文で最初に受信したデータを処理
- タイムアウト処理も実装

### []課題6: worker poolパターン
固定数のworkerで大量のタスクを処理するプログラムを作成してください。
- 3つのworker goroutineを起動
- 100個のタスクを分散処理
- すべてのタスク完了を確認

### []課題7: contextを使ったキャンセル処理
`context.Context`を使ってgoroutineをキャンセルできるプログラムを作成してください。
- 長時間実行されるgoroutineを作成
- 一定時間後にcontextをキャンセル
- goroutineが適切に終了することを確認

### []課題8: errgroup.Groupを使ったエラーハンドリング
`golang.org/x/sync/errgroup`を使って、複数のgoroutineのエラーを適切に処理するプログラムを作成してください。
- 複数のgoroutineで並行処理を実行
- いずれかのgoroutineでエラーが発生したら全体を中断
- 最初に発生したエラーを返す

### []課題9: Mutexを使った排他制御
複数のgoroutineから共有データに安全にアクセスするプログラムを作成してください。
- `sync.Mutex`を使用
- 10個のgoroutineがカウンタをインクリメント
- race conditionが発生しないことを確認（`go run -race`で検証）

### []課題10: チャネルのクローズとrange
チャネルを適切にクローズし、rangeで受信するプログラムを作成してください。
- 送信側がチャネルをクローズ
- 受信側が`range`で全データを受信
- クローズ後の動作を確認

### []課題11: 並行処理のパフォーマンス測定
順次処理と並行処理のパフォーマンスを比較するプログラムを作成してください。
- 重い計算処理を複数回実行
- 順次実行と並行実行の時間を測定
- 結果を比較して出力

### []課題12: semaphoreパターン
バッファ付きチャネルを使って、同時実行数を制限するプログラムを作成してください。
- 最大3つまで同時実行
- 10個のタスクを順次処理
- セマフォの動作を確認

### []課題13: sync.Onceを使った初期化
`sync.Once`を使って、初期化処理を一度だけ実行するプログラムを作成してください。
- 複数のgoroutineから初期化関数を呼び出し
- 実際には一度だけ実行されることを確認
- シングルトンパターンの実装

### []課題14: sync.RWMutexを使った読み書きロック
読み取りと書き込みで異なるロックを使うプログラムを作成してください。
- `sync.RWMutex`を使用
- 複数の読み取りgoroutineと少数の書き込みgoroutine
- 読み取りロックは同時実行可能なことを確認

### []課題15: fan-out/fan-inパターン
1つの入力を複数のgoroutineに分散し、結果を集約するプログラムを作成してください。
- 1つのチャネルから複数のworkerに分散（fan-out）
- 複数のworkerの結果を1つのチャネルに集約（fan-in）
- すべての結果を収集

### []課題16: pipelineパターン
チャネルを使ってデータ処理パイプラインを構築するプログラムを作成してください。
- 複数のステージ（生成→変換→フィルタ→出力）
- 各ステージを別のgoroutineで実装
- チャネルでステージ間を接続

### []課題17: done channelパターン
done channelを使って終了シグナルを複数のgoroutineに伝播するプログラムを作成してください。
- done channelをクローズして終了を通知
- 複数のgoroutineが終了シグナルを受け取る
- 適切にクリーンアップ処理を実行

### []課題18: rate limitingパターン
`time.Ticker`を使ってリクエストのレート制限を実装するプログラムを作成してください。
- 1秒間に最大3リクエストまで
- それ以上は待機させる
- レート制限の動作を確認

### []課題19: context.WithTimeoutを使ったタイムアウト制御
`context.WithTimeout`を使って処理にタイムアウトを設定するプログラムを作成してください。
- 長時間かかる可能性のある処理を実行
- タイムアウト時間を設定
- タイムアウト発生時に適切に処理を中断
